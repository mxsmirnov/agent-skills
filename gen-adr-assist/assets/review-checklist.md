# Checklist для review короткого ADR

Проверь по порядку:

1. [ ] Сформулировано ли решение одной фразой?
2. [ ] Понятны ли границы решения (scope)?
3. [ ] Есть ли не меньше двух реальных альтернатив?
4. [ ] Понятны ли decision drivers и ограничения?
5. [ ] Не противоречит ли решение `tech-radar.json`?
6. [ ] Если технология указана явно, проверен ли её ring?
7. [ ] Если технология отсутствует в радаре, задан ли вопрос о её добавлении?
8. [ ] Есть ли заметные негативные последствия, а не только преимущества?
9. [ ] Имя файла: adr-NNNN-kebab-case.md
